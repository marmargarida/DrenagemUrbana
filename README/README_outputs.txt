1. Context
----------
This folder contains all processed outputs generated by the Python
scripts developed for the analysis, validation, and consolidation of
urban drainage scenarios simulated in SWMM. Each script reads the
original simulation files (.inp and .rpt) and produces structured
datasets, audit reports, statistical summaries, and plots.

2. Outputs by script
--------------------

(a) scenarios_peak_depth_analysis.py
------------------------------------
Generates outputs for a single scenario:
- scenarioXX_peak_statistics.csv
  Peak depth statistics (maximum, minimum, mean, standard deviation).
- scenarioXX_depth_timeseries.png
  Time series plot of water depth (m) in five selected nodes:
  the node with the highest peak, the lowest peak, and three intermediates.

(b) scenarios_global_peak_analysis.py
-------------------------------------
Generates outputs across all scenarios:
- scenarioXX_peak_statistics.csv
  Individual statistics for each scenario.
- scenarioXX.pdf
  Individual time series plots of selected nodes.
- all_scenarios_peak_statistics.csv
  Combined table with peak depth statistics from all scenarios.
- all_scenarios_max_curves.pdf
  Comparative plot showing maximum depth curves across all scenarios.

(c) scenarios_input_audit.py
----------------------------
Audits the input CSVs of each scenario:
- scenarioXX_audit_report.txt
  Audit reports summarizing:
  * First value per variable
  * Number of non-empty values
  * Number of empty cells
  * Measurement units

(d) scenarios_data_extractor.py
-------------------------------
Extracts and consolidates hydrological and hydraulic variables:
- scenarioXX.csv
  Structured dataset with parameters from .inp and .rpt files.
- scenarioXX.parquet
  Same dataset as CSV, saved in Apache Parquet format for optimized analysis.
- scenarioXX_audit_report.txt
  Audit reports of the datasets, with summary of variables, values, and units.

3. Notes
--------
- All depths are expressed in meters (m).
- Time series plots use hh:mm time format.
- CSV and Parquet datasets include units for clarity.
- Audit reports ensure data transparency and traceability.
- Outputs are suitable for direct use in scientific publications,
  reproducibility studies, and advanced modeling.
